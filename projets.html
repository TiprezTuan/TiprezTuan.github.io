Projet de conception d'un sudoku en C



Dans le cadre de la SAE 1.01, nous avons réalisé un programme de Sudoku en langage C, capable de s'exécuter dans une console.

Pour réaliser ce projet, nous avons suivi plusieurs étapes :

  1. Conception de la maquette : Nous avons d'abord conçu une maquette du projet pour définir son architecture et son fonctionnement.

  2. Développement : Ensuite, nous avons développé une première version du programme en implémentant les fonctionnalités de base.

  3. Élaboration d'un cahier de tests : Après avoir créé la première version, nous avons élaboré un cahier de tests pour identifier et corriger les problèmes.

  4. Vérification et validation : Nous avons utilisé le cahier de tests pour vérifier que notre code répond pleinement aux exigences du client.

  5. Création d'un fichier de traces d'exécutions : Nous avons généré un fichier montrant les différentes traces d'exécutions pour documenter le comportement du programme et vérifier son bon fonctionnement.

  6. Réalisation de la documentation avec Doxygen : Enfin, nous avons créé la documentation du projet en utilisant Doxygen, afin de fournir une description détaillée et structurée du code et de ses fonctionnalités.

Création d'un résolveur de sudoku en C



Dans le cadre de la SAE 1.02, nous avons réalisé en langage C, des programmes de résolution de Sudoku en implémentant plusieurs algorithmes : backtracking, paires nues, singleton caché et singleton nu.

Cela dans le but de comparer les différents algorithmes.

  1. Backtracking : Essaye toutes les possibilités en remplissant les cases une par une, revenant en arrière en cas de conflit.
  2. Paires nues : Identifie deux cases dans une ligne, colonne ou boîte contenant les mêmes deux candidats, éliminant ces candidats des autres cases.
  3. Singleton caché : Trouve un candidat unique pour une case dans une ligne, colonne ou boîte, éliminant les autres candidats pour cette case.
  4. Singleton nu : Identifie une case avec un seul candidat possible et l'assigne directement.
